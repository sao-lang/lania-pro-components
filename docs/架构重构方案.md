# Pro 组件架构重构方案

## 概述

本方案针对 `ProForm`、`ProDialog`、`ProTable` 三个组件中存在的架构问题进行系统性重构，核心目标是**消除重复实例、明确职责边界、统一渲染路径**。

---

## 一、问题全景

### 1.1 三个组件的共性问题

| 组件          | `useXxx` 存在      | `Xxx` 调用 `useXxx`        | 问题类型                                                                                                  |
| ------------- | ------------------ | -------------------------- | --------------------------------------------------------------------------------------------------------- |
| **ProForm**   | `useProForm.tsx`   | `ProForm.tsx` 内部调用     | **重复实例**：用户 `useProForm()` + `<ProForm>` → 两个 FormStore                                          |
| **ProDialog** | `useProDialog.tsx` | `ProDialog.tsx` **不调用** | **重复实现**：`useProDialog` 内有完整的 `InternalDialog`，与 `ProDialog.tsx` 完全独立的 Modal/Drawer 渲染 |
| **ProTable**  | `useProTable.tsx`  | `index.tsx` 内部调用       | **无效调用**：传入了 store 不会创建重复实例，但创建的 instance/ref 从未被使用                             |

### 1.2 每个组件的详细问题

#### ProForm

```
用户代码                          ProForm 组件
┌───────────────┐               ┌──────────────────────────┐
│ useProForm()  │               │  ProForm (forwardRef)    │
│               │               │                          │
│ instance A ───┼── 传入？ ──►  │  useProForm()  ← instance B
│ formStore A   │               │  formStore B             │
└───────────────┘               │  ═══ 两份独立状态 ═══    │
```

**根因**：`ProForm.tsx` 无条件调用 `useProForm()`，没有检测外部是否已创建实例。
**范围**：仅 `useProForm` + `ProForm` 组合使用时触发。
**方案**：`form` prop 注入模式（详见二）。

---

#### ProDialog

```
独立实现 1：ProDialog.tsx              独立实现 2：useProDialog.tsx InternalDialog
┌─────────────────────────┐           ┌──────────────────────────────┐
│  createProDialogComponent│           │  InternalDialog              │
│                         │           │                              │
│  useDialogInstance()    │           │  useDialogInstance()  ← 相同 │
│                         │           │                              │
│  if (drawer) <Drawer/>  │           │  if (drawer) <Drawer/>  ← 复制│
│  else        <Modal/>   │           │  else        <Modal/>  ← 复制│
│                         │           │                              │
│  closeDialog: () => {}   │           │  closeDialog: handleClose    │
└─────────────────────────┘           └──────────────────────────────┘
```

**根因**：`useProDialog` 为了实现命令式调用，在 `createImperativeInstance` 回调中自行创建了一个 `InternalDialog` 组件，它和 `ProDialog.tsx` 做了完全一样的事情。

**具体问题**：

1. **~200 行重复 JSX**：Modal 和 Drawer 的渲染代码在两个文件中完全一致
2. **`closeDialog` 行为不一致**：`ProDialog.tsx` 传 `() => {}`（空操作），`InternalDialog` 传 `handleClose`（真实关闭）
3. **`InternalDialog` 同时创建 Drawer 和 Modal**：使用 `if (mode === 'drawer')` 条件分支，每次渲染都会创建完整的条件判断路径，两个组件都在内存中实例化
4. **循环依赖**：`ProDialog.tsx` → `dialogHolder.tsx` → `index.tsx` → `ProDialog.tsx`
5. **`useDialogInstance` 超级 hook**：400 行、55 字段入参、混合返回渲染函数和状态逻辑

---

#### ProTable

```
ProTableComponent（index.tsx）
│
├── const store = useMemo(() => createDataStore(), [])
├── useRequest({ store, ... })
├── useProTable({ store, ... })  ← 传入了 store，不会重复创建
│   └── defaultStoreRef 分支跳过
│   └── 创建了 tableRef + instance，但从未使用
│   └── 创建了 Provider，但也从未使用
├── useEditableTable({ store, ... })
│
└── <RootProvider>
      <DataProvider store={store} action={action}>
        ...
      </DataProvider>
    </RootProvider>
```

**根因**：`useProTable` 在 `ProTableComponent` 内部的调用是无意义的——它创建的 `tableRef`、`instance`、`Provider` 都没有被消费。组件使用自己的 `action` 对象暴露实例。

**具体问题**：

1. `useProTable` 调用产生未使用的 `tableRef` 和 `Provider`（内存 + 渲染浪费）
2. 类型转换问题：`store as unknown as DataStoreImpl<Record<string, unknown>>`
3. `useProTable` 和 `ProTableComponent` 内部各自的 Context 体系并存

## 二、ProForm 改造方案

### 2.1 目标

- 消除 `useProForm()` + `<ProForm>` 的重复实例
- 公开 API 不变（零破坏性变更）
- 内部职责清晰：数据层 vs UI 层

### 2.2 改动文件

| 文件                     | 改动                                                                                          |
| ------------------------ | --------------------------------------------------------------------------------------------- |
| `ProForm/types.ts`       | 新增 `ProFormProps.form: UseProFormReturn`                                                    |
| `ProForm/useProForm.tsx` | 内部分拆 `useFormStore` + `useFormUI`，**不再返回 Provider**                                  |
| `ProForm/ProForm.tsx`    | 新增 `form` prop 检测，分派到 `ProFormWithForm` / `ProFormStandalone`，提取 `ProFormRenderer` |

### 2.3 架构

```
useFormStore(schemas)           useFormUI(schemas, formStore, arcoForm)
  (内部，纯数据层)                (内部，UI 能力层)
  ┌──────────────────┐          ┌─────────────────────┐
  │ formStore         │          │ fieldNavigation     │
  │ arcoForm          │          │ fieldStatusMap      │
  │ baseInstance:     │          │ virtualScroll       │
  │  validate         │          │ scrollToField       │
  │  setFieldsValue   │          │ focusField          │
  │  getFieldsValue   │          │ setFieldStatus      │
  │  resetFields      │          │ draft/preview state │
  │  submit           │          └─────────────────────┘
  └──────────────────┘                    │
         │                                │
         └────────────┬───────────────────┘
                      ▼
            useProForm（公开 API）
            ┌─────────────────────┐
            │ instance = 数据+UI   │
            │ bindingProps         │
            └─────────────────────┘
                      │ form prop
                      ▼
            ProForm（调度器）
            ┌─────────────────────┐
            │ 有 form → 复用       │
            │ 无 form → 内部创建   │
            └─────────────────────┘
                      │
                      ▼
            ProFormRenderer（纯渲染）
            ┌─────────────────────┐
            │ Schema 合并          │
            │ Grid 布局 / 按钮     │
            │ 草稿持久化           │
            │ Context 包裹         │
            └─────────────────────┘
```

### 2.4 使用方式

```tsx
// 方式 A（推荐，编程式控制）
const form = useProForm({ schemas });
return <ProForm form={form} columns={2} />;
// form.instance 是唯一的

// 方式 B（简单场景）
const ref = useRef(null);
return <ProForm ref={ref} schemas={[...]} />;
// ref.current 获取实例
```

## 三、ProDialog 改造方案

### 3.1 目标

- 消除 `InternalDialog` 与 `ProDialog.tsx` 的重复实现
- `useProDialog` 复用 `ProDialog` 组件而非自行渲染
- 消除循环依赖
- 消除 `closeDialog` 不一致

### 3.2 核心思路

**`useProDialog` 不再创建 `InternalDialog`，改为命令式渲染 `ProDialog` 组件。**

```
改造前：
useProDialog → createImperativeInstance → InternalDialog → useDialogInstance → Modal/Drawer
ProDialog    → createProDialogComponent → useDialogInstance → Modal/Drawer

改造后：
useProDialog → createImperativeInstance → ProDialog → useDialogInstance → Modal/Drawer
                                              ↑
                                        (唯一渲染路径)
```

### 3.3 改动文件

| 文件                              | 改动                                                                                                   |
| --------------------------------- | ------------------------------------------------------------------------------------------------------ |
| `ProDialog/ProDialog.tsx`         | 新增 `__internal` prop 支持命令式调用；暴露实例给外部；修复 `closeDialog: () => {}` 为真实关闭         |
| `ProDialog/useProDialog.tsx`      | **删除** `InternalDialog`；改为用 `createImperativeInstance` 渲染 `ProDialog`；移除循环依赖            |
| `ProDialog/useDialogInstance.tsx` | 将渲染函数（`renderTitle`/`renderFooter`/`renderContent`）抽离回 `ProDialog.tsx`，保持 hook 仅处理逻辑 |
| `ProDialog/dialogHolder.tsx`      | 修复循环依赖，改为动态 `import()` 或共用类型                                                           |
| `ProDialog/index.tsx`             | 更新导出                                                                                               |

### 3.4 详细改造

#### 3.4.1 `ProDialog.tsx` — 新增受控 + 实例暴露

```tsx
// ProDialog.tsx

interface ProDialogProps {
  // ... 现有 props 不变
  /** @internal 命令式实例的回调 */
  __onInstanceReady?: (instance: ProDialogInstance) => void;
}

const ProDialogComponent = forwardRef((props, ref) => {
  const { __onInstanceReady, ...rest } = props;

  const [visible, setVisible] = useState(controlled ? props.visible : false);
  const dialogInstance = useMemo(() => createDialogInstance(...), [...]);

  // 暴露实例给外部（包括 useProDialog 的 createImperativeInstance）
  useImperativeHandle(ref, () => dialogInstance, [dialogInstance]);
  useEffect(() => { __onInstanceReady?.(dialogInstance); }, [dialogInstance]);

  // 修复 closeDialog: 改为真实关闭
  const { ... } = useDialogInstance({
    ...rest,
    closeDialog: () => setVisible(false),  // ← 从 () => {} 改为真实关闭
  });

  return (
    <>
      {mode === 'drawer' ? <Drawer .../> : <Modal .../>}
    </>
  );
});
```

#### 3.4.2 `useProDialog.tsx` — 复用 ProDialog 组件

```tsx
// useProDialog.tsx — 改造后

export function useProDialog(options) {
  const { name, fullscreenProp, ...dialogProps } = options;
  const instanceRef = useRef(null);
  const managerRef = useRef(null);

  const ensureManager = useCallback(() => {
    if (!managerRef.current) {
      managerRef.current = createImperativeInstance(({ dialogProps, params }) => (
        <ProDialog
          {...dialogProps}
          __onInstanceReady={(instance) => {
            instanceRef.current = instance;
            instance.open(params);
          }}
        />
      ));
    }
    return managerRef.current;
  }, []);

  const open = useCallback(
    (params) => {
      const manager = ensureManager();
      manager.open({ dialogProps, params });
    },
    [dialogProps],
  );

  const close = useCallback(() => {
    instanceRef.current?.close();
  }, []);

  // ... 其余方法
}
```

#### 3.4.3 `useDialogInstance.tsx` — 纯逻辑 hook

将以下内容从 hook 移回 `ProDialog.tsx`：

- `renderTitle` / `renderTitleWithDrag`
- `renderFooter`
- `renderContent`

hook 仅保留：

- `dialogInstance` 创建
- `handleOk` / `handleClose` 逻辑
- `computedWidth` / `computedHeight` 计算
- `dragOffset` / `resizeSize` 拖拽状态
- `toggleFullscreen`

```tsx
// useDialogInstance.tsx — 改造后，不再返回渲染函数
export function useDialogInstance(options) {
  // 仅状态逻辑
  return {
    dialogInstance,
    handleOk,
    handleClose,
    computedWidth,
    computedHeight,
    dragOffset,
    resizeSize,
    toggleFullscreen,
  };
  // ✗ 不再返回 renderTitle / renderFooter / renderContent
}
```

## 四、ProTable 改造方案

### 4.1 目标

- **对齐 ProForm 模式**：`useProTable` = 纯状态层，`ProTable` = 纯渲染层
- `ProTable` 新增 `table` prop，有则直接渲染，无则内部调 `useProTable`
- `useProTable` **接管**目前 `ProTableComponent` 内部的所有状态 hooks
- 消除类型转换问题

### 4.2 当前现状

```
ProTableComponent（index.tsx 内）
┌─────────────────────────────────────────────┐
│ 状态层（散布在组件内部）                         │
│  const store = createDataStore()              │
│  useRequest({ store })          ← 请求状态    │
│  useEditableTable({ store })    ← 编辑状态    │
│  useDragSort(dataSource)        ← 拖拽状态    │
│  useProTable({ store })         ← 实例状态    │
│                                              │
│  action = { reload, reset, ... }  ← 实例方法 │
│                                              │
│ 渲染层（交织在状态层中）                         │
│  useVirtualScroll()              ← UI 优化    │
│  Context 三层包裹              ← UI 渲染      │
│  TableRenderer / Toolbar / ... ← UI 渲染      │
└─────────────────────────────────────────────┘

useProTable（独立使用时）
┌─────────────────────────────────────────────┐
│ DataStore + instance + bindingProps + Provider│
│ 不包含：useRequest / useEditable / useDragSort│
└─────────────────────────────────────────────┘
```

### 4.3 改造后架构

```
useProTable（纯状态层 — 接管全部状态 hooks）
┌─────────────────────────────────────────────┐
│ DataStore                                    │
│ useRequest({ store })          ← 从组件移入  │
│ useEditableTable({ store })    ← 从组件移入  │
│ useDragSort(dataSource)        ← 从组件移入  │
│                                              │
│ store / action / dataSource / loading        │
│ pagination / selectedRows / query            │
│                                              │
│ ✗ Provider 不再返回（由 ProTable 内部创建）    │
│ ✗ useVirtualScroll 不移入（纯 UI 优化）       │
└─────────────────────────────────────────────┘
                      │ table prop
                      ▼
ProTable（纯渲染层 — 新增 table prop）
┌─────────────────────────────────────────────┐
│ 有 table → 复用外部状态，跳过 useProTable    │
│ 无 table → 内部调 useProTable（一次）        │
│                                              │
│ Provider 内部创建（Root/Data/Column 三层）    │
│ useVirtualScroll                 ← UI 优化   │
│ TableRenderer / Toolbar / Pagination         │
│ BatchOperation / CardView / ViewModeSwitch   │
└─────────────────────────────────────────────┘
```

### 4.4 改动文件

| 文件                             | 改动                                                                                                                          |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `ProTable/types.ts`              | 新增 `ProTableProps.table: UseProTableReturn`                                                                                 |
| `ProTable/hooks/useProTable.tsx` | **扩展**：加入 `useRequest` / `useEditableTable` / `useDragSort`；**移除** `Provider` 返回；补齐 store、action 等内部字段导出 |
| `ProTable/index.tsx`             | **大幅精简**：移除所有状态 hooks 调用，改为 `table` prop 分发 + 内部创建 `ProTableRenderer` 纯渲染组件                        |

### 4.5 详细改造

#### 4.5.1 `useProTable.tsx` — 扩展为完整状态层

```tsx
// useProTable.tsx — 改造后

export interface UseProTableReturn<T = Record<string, unknown>> {
  /** DataStore */
  store: DataStoreImpl<T>;
  /** 实例方法（reload, reset, getPagination, getSelectedRows, 等） */
  instance: ProTableInstance;
  /** 可绑定到 ProTable 的 props */
  bindingProps: ProTableProps<T>;
  /** 数据源 */
  dataSource: T[];
  /** 加载状态 */
  loading: boolean;
  /** 分页信息 */
  pagination: { current: number; pageSize: number };
  /** 选中行 keys */
  selectedRowKeys: (string | number)[];
  /** 选中行数据 */
  selectedRows: T[];
  /** 查询参数 */
  query: Record<string, unknown>;
  /** 可编辑实例（供 ProTable 内部使用） */
  editableInstance: EditableTableInstance;
  /** 拖拽排序后的数据源（供 ProTable 内部使用） */
  dragSortedDataSource: T[];
  /** 拖拽行 props */
  getDragRowProps: (record: T) => Record<string, unknown>;
  /** 拖拽手柄 props */
  getDragHandleProps: (record: T) => Record<string, unknown>;
  /** 重置拖拽排序 */
  resetDragSort: () => void;
  /** 查询表单 ref（供 ProTable 内部使用） */
  formRef: React.RefObject<ProFormInstance>;
  /** 查询表单 ref（供 ProTable 内部使用） */
  queryFormRef: React.RefObject<ProQueryFormInstance>;
  /** 展开行 keys */
  expandedRowKeys: (string | number)[];
  /** 设置展开行 */
  setExpandedRowKeys: (keys: (string | number)[]) => void;
  /** 轮询控制 */
  startPolling: () => void;
  stopPolling: () => void;
  /** 缓存 */
  cacheHookResult: ReturnType<typeof useCache>;
  /** ✗ Provider 不再返回 */
}

export const useProTable = <T>(options: UseProTableOptions<T>): UseProTableReturn<T> => {
  const {
    store: propStore,           // 外部传入 store → 复用；不传 → 内部创建
    columns, request, toolbar, search, batchOperation,
    editable, dragSort, cache, cacheKey,
    rowKey, defaultPageSize, pageSizeOptions,
    manual, debounceTime, polling,
    beforeRequest, afterRequest, onRequestError, postData,
    ...rest
  } = options;

  // ===== 数据层 =====
  const store = useMemo(() => propStore ?? createDataStore<T>({ ... }), [propStore]);
  const formRef = useRef<ProFormInstance>(null);
  const queryFormRef = useRef<ProQueryFormInstance>(null);

  // ===== 请求层 =====
  const cacheHook = useCache<{ data: T[]; total: number }>({ ... });
  const { fetchData, startPolling, stopPolling } = useRequest<T>({
    store, request, manual, debounceTime, polling,
    beforeRequest, afterRequest, onRequestError, postData,
    cache: cache ? cacheHook : undefined, cacheKey,
  });

  // ===== 编辑层 =====
  const { instance: editableInstance, ...editableMethods } = useEditableTable<T>({
    config: editable, dataSource: store.dataSource, getRowKey,
  });

  // ===== 拖拽层 =====
  const { sortedDataSource: dragSortedDataSource, getDragRowProps, getDragHandleProps, resetSort: resetDragSort }
    = useDragSort<T>({ dataSource: store.dataSource, config: dragSort, enabled: !!dragSort, getRowKey });

  // ===== 展开行 =====
  const [expandedRowKeys, setExpandedRowKeys] = useState(...);

  // ===== 实例方法 =====
  const instance = useMemo<ProTableActionType<T>>(() => ({
    reload: (resetPageIndex) => { ... },
    reset: () => { formRef.current?.resetFields(); store.reset(); store.reload(); },
    getSelectedRows: () => store.selectedRows,
    getPagination: () => ({ current: store.page, pageSize: store.pageSize, total: store.total }),
    startEditable: editableMethods.startEditable,
    cancelEditable: editableMethods.cancelEditable,
    saveEditable: editableMethods.saveEditable,
    deleteEditable: editableMethods.deleteEditable,
    startPolling, stopPolling,
    // ... 所有其他方法
  }), [store, editableMethods, fetchData, startPolling, stopPolling, ...]);

  // ===== bindingProps（直接供 ProTable 渲染）=====
  const bindingProps = useMemo<ProTableProps<T>>(() => ({
    columns, dataSource: store.dataSource, request, toolbar, search, batchOperation,
    pagination: propPagination === false ? false : { ... },
    cardContainer, urlSync, searchSchema, editable, dragSort, cache, cacheKey,
    rowKey, defaultPageSize, pageSizeOptions, loading: store.loading,
    emptyRender, errorRender, beforeRequest, afterRequest, onRequestError, postData,
    debounceTime, polling, manual,
  }), [...]);

  return {
    store, instance, bindingProps,
    dataSource: store.dataSource, loading: store.loading,
    pagination: store.pagination, selectedRowKeys: store.selectedRowKeys,
    selectedRows: store.selectedRows, query: store.query,
    editableInstance, dragSortedDataSource, getDragRowProps, getDragHandleProps, resetDragSort,
    formRef, queryFormRef, expandedRowKeys, setExpandedRowKeys,
    startPolling, stopPolling, cacheHookResult: cacheHook,
    // ✗ 不再返回 Provider
  };
};
```

#### 4.5.2 `ProTable` — 精简为渲染层

```tsx
// index.tsx — 改造后

const ProTableRenderer: React.FC<ProTableRendererProps> = (props) => {
  // 从 props 接收完整状态，不再调用任何状态 hooks
  const {
    store, instance, dataSource, loading, selectedRowKeys, selectedRows,
    editableInstance, dragSortedDataSource, getDragRowProps, getDragHandleProps,
    formRef, queryFormRef, expandedRowKeys, setExpandedRowKeys,
    cacheHookResult,
    // 原始 UI props
    columns, request, toolbar, search, batchOperation,
    pagination, cardContainer, ...uiProps
  } = props;

  // ===== 仅 UI 相关 hooks =====
  const { state: virtualScrollState, containerRef: vsContainerRef, onScroll, scrollToIndex, ... }
    = useVirtualScroll<T>(dragSortedDataSource, { enabled: !!virtualScroll, ... });

  const [viewMode, setViewMode] = useState<'table' | 'card'>(propViewMode || 'table');

  // ===== 事件处理器 =====
  const action = useMemo(() => ({
    ...instance,
    scrollToIndex, scrollToTop: scrollToTopVirtual, scrollToBottom: scrollToBottomVirtual,
    clearCache: cacheHookResult.clearCache,
    openDialog: (config) => openDialog({ ...dialogConfig?.open, ...config }),
    confirm: (config) => confirm({ ...dialogConfig?.confirm, ...config }),
  }), [instance, scrollToIndex, ...]);

  // ===== 渲染 =====
  return (
    <RootProvider props={props}>
      <DataProvider store={store} formRef={formRef} action={instance}>
        <ColumnProvider initialColumns={columns} ...>
          {cardContainer ? <Card>...</Card> : <div>...</div>}
          {/* QueryForm / Toolbar / BatchOperation / TableRenderer / Pagination / CardView */}
        </ColumnProvider>
      </DataProvider>
    </RootProvider>
  );
};

// 调度层
const ProTableComponent = forwardRef((props, ref) => {
  if (props.table) {
    // 方式 A：外部注入状态 → 纯渲染
    return <ProTableRendererWithRef {...props} table={props.table} ref={ref} />;
  }
  // 方式 B：内部创建状态
  return <ProTableStandalone {...props} ref={ref} />;
});

const ProTableStandalone = forwardRef((props, ref) => {
  const state = useProTable(props);
  useImperativeHandle(ref, () => state.instance, [state.instance]);
  return <ProTableRenderer {...props} {...state} />;
});

const ProTableRendererWithRef = forwardRef((props, ref) => {
  const { table: state } = props;
  useImperativeHandle(ref, () => state.instance, [state.instance]);
  return <ProTableRenderer {...props} {...state} />;
});

export const ProTable = ProTableComponent as ProTableComponentType;
```

### 4.6 使用方式

```tsx
// 方式 A（推荐，编程式控制）
const table = useProTable({ columns, request });
return <ProTable table={table} />;
// table.instance 是唯一的

// 方式 B（简单场景）
const ref = useRef(null);
return <ProTable ref={ref} columns={...} request={...} />;
// ref.current 获取实例
```

### 4.7 与 ProForm 的对比

| 维度            | ProForm 改造                         | ProTable 改造                                             |
| --------------- | ------------------------------------ | --------------------------------------------------------- |
| 状态 hook       | `useProForm`                         | `useProTable`                                             |
| 注入 prop       | `form`                               | `table`                                                   |
| 组件            | `ProForm`                            | `ProTable`                                                |
| Provider 创建   | 组件内部                             | 组件内部                                                  |
| 内部 hooks 迁移 | `useFormStore` + `useFormUI`（拆分） | `useRequest` / `useEditableTable` / `useDragSort`（移入） |
| 改动性质        | 拆分                                 | 合并                                                      |
