# Go Rules

- 错误处理：始终检查 `err != nil`，不忽略返回值
- 使用 `context.Context` 作为函数第一个参数（需要时）
- 接口定义保持小粒度（1-2 方法）
- 使用 `go fmt` 格式化代码，遵循 Go 标准命名风格
- 避免 `interface{}`，优先用 `any`

## Testing

- 使用 `testing` 标准库编写测试，函数名 `TestXxx`
- 测试文件命名 `*_test.go`，与被测文件同包
- 使用 `t.Run()` 组织子测试
- 覆盖正常路径、边界条件和错误路径
- Table-driven tests 是 Go 推荐模式

## Lint

- 运行 `go fmt ./...` 格式化代码
- 运行 `go vet ./...` 静态分析
- 运行 `golangci-lint run` 全面检查
