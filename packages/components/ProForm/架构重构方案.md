# ProForm 架构重构方案 — behavior → status 流水线

## 一、改造目标

1. **schema.behavior**：从 `{ visible, disabled, readonly }` 三个散装 boolean 改为 `FieldStatus | ((values) => FieldStatus) | undefined`，直接声明字段想要的最终态
2. **表单级状态**：`<ProForm preview readonly disabled />` 优先级高于字段级
3. **流水线统一**：`behavior decl → resolve → merge form constraints → effectiveStatus`
4. **`draft`** 从状态中移除，只作为持久化配置，仅在 `edit` 态生效
5. **自定义表单控件**：接收聚合 props `{ value, onChange, status, values, schema, field, form }`
6. **自定义只读渲染器**：签名加第 5 参数 `meta: { status, values }`
7. **删除废弃类型**：`FieldBehavior`, `ComputedFieldBehavior`

## 二、类型定义变更（types.ts）

### 新增

```typescript
type FieldStatus = 'edit' | 'readonly' | 'disabled' | 'hidden';
type BehaviorDecl = FieldStatus | ((values: Record<string, unknown>) => FieldStatus) | undefined;
```

### 删除

- `interface FieldBehavior` — 替换为 `BehaviorDecl`
- `interface ComputedFieldBehavior` — 不再需要，直接使用 `FieldStatus`
- `FormStatus` 中的 `'draft'` 枚举值

### 修改

- `ProFormSchema.behavior` 类型从 `FieldBehavior` → `BehaviorDecl`
- `FieldNodeAPI` 删除 `computedBehavior` 和 `updateComputedBehavior`
- `FieldContextValue` 删除 `computedBehavior`
- `ReadonlyRenderer` 签名加第 5 参数 `meta: { status: FieldStatus; values: Record<string, unknown> }`

## 三、核心流水线（FieldNode.ts）

### 删除

- `computeBehaviorValue()` 函数
- `ComputedFieldBehavior` 接口
- `_computedBehavior: ComputedRef<ComputedFieldBehavior>` 字段
- `get computedBehavior()` getter
- `updateComputedBehavior()` 方法
- `updateStatusFromBehavior()` 方法
- 对应的 `watch(_computedBehavior.value)` 监听

### 新增

```typescript
_effectiveStatus: ComputedRef<FieldStatus>;

// 在 constructor 中：
this._effectiveStatus = computed<FieldStatus>(() => {
  const values = store.getValues();
  const formConstraints = store.getFormConstraints(); // { preview, readonly, disabled }

  // 1. 解析 schema.behavior
  let fieldWanted: FieldStatus | undefined;
  const behavior = schema.behavior as BehaviorDecl;
  if (behavior === undefined) {
    fieldWanted = undefined;
  } else if (typeof behavior === 'function') {
    fieldWanted = behavior(values);
  } else {
    fieldWanted = behavior;
  }

  // 2. 合并：全局优先级高于字段级
  if (fieldWanted === 'hidden') return 'hidden';
  if (formConstraints.preview) return 'readonly';
  if (formConstraints.readonly) return 'readonly';
  if (formConstraints.disabled) return 'disabled';
  return fieldWanted ?? 'edit';
});
```

### 状态同步

- FieldNode 的 `_status` ref 通过 watch `_effectiveStatus` 自动同步：

```typescript
watch(
  () => this._effectiveStatus.value,
  (newStatus, oldStatus) => {
    if (newStatus !== oldStatus) {
      this.setStatus(newStatus);
    }
  },
  { immediate: true },
);
```

## 四、FormStore 变更

### 新增

```typescript
private formConstraints = reactive<{ preview: boolean; readonly: boolean; disabled: boolean }>({
  preview: false, readonly: false, disabled: false,
});

setFormConstraints(constraints: { preview?: boolean; readonly?: boolean; disabled?: boolean }): void {
  Object.assign(this.formConstraints, constraints);
}

getFormConstraints(): { preview: boolean; readonly: boolean; disabled: boolean } {
  return { ...this.formConstraints };
}
```

### 删除

- `field.updateComputedBehavior(this.getValues())` 调用（在依赖监听中）

## 五、RootContext 变更

### FormState 去掉 draft

```typescript
interface FormState {
  readonly: boolean;
  disabled: boolean;
  preview: boolean;
  submitting: boolean;
  status: FormStatus; // 'edit' | 'preview' | 'readonly' | 'disabled'
}
```

### createFormState 去掉 draft 参数

```typescript
function createFormState(preview, readonly, disabled, submitting): FormState {
  let status: FormStatus = 'edit';
  if (preview) status = 'preview';
  else if (readonly) status = 'readonly';
  else if (disabled) status = 'disabled';
  return { readonly, disabled, preview, submitting, status };
}
```

## 六、FieldContext 变更

### 删除

- `computedBehavior: { visible, disabled, readonly }` 字段
- `defaultFieldContext` 中对应的默认值

## 七、FormField 变更

### 自定义表单控件渲染

从散装 props 改为聚合传：

```tsx
// 改造前
<ComponentToRender
  disabled={status === 'disabled'}
  visible={fieldNode.computedBehavior.visible}
  readonly={fieldNode.computedBehavior.readonly}
  placeholder={...} options={...} value={...}
  {...restComponentProps}
/>

// 改造后
<ComponentToRender
  value={componentValue}
  onChange={handleChange}
  status={status}
  values={rootContext.instance.getFieldsValue()}
  schema={resolvedSchema}
  field={fieldNode}
  form={rootContext.instance}
  {...restComponentProps}
/>
```

### 只读渲染器调用

```tsx
// 改造前
renderer(displayValue, resolvedSchema.options, readonlyConfig, resolvedSchema.componentProps);

// 改造后
renderer(displayValue, resolvedSchema.options, readonlyConfig, resolvedSchema.componentProps, {
  status,
  values: rootContext.instance.getFieldsValue(),
});
```

### FieldContext 构建

删除 `computedBehavior`，保留 `status` 即可

### hidden 状态判断

```tsx
// 改造前
if (status === 'hidden') return null;
display: fieldNode.computedBehavior.visible ? undefined : 'none';

// 改造后
if (status === 'hidden') return null;
// outer div 不再需要 display:none，因为 hidden 已在顶部返回
```

### status 判断更新

```tsx
// 改造前
if (status === 'preview' || status === 'readonly') {
  return renderReadonlyContent;
}

// 改造后
if (status === 'readonly') {
  return renderReadonlyContent;
}
// preview 已合并为 readonly，不再需要单独判断
```

## 八、ProForm 变更

### formState 创建

```tsx
// 改造前
const formState = createFormState(isDraftState, readonly, disabled, isPreviewState, submitLoading);

// 改造后
const formState = createFormState(isPreviewState, readonly, disabled, submitLoading);
```

### 同步 formConstraints 到 store

```tsx
useEffect(() => {
  formStore.setFormConstraints({
    preview: isPreviewState,
    readonly,
    disabled,
  });
}, [isPreviewState, readonly, disabled, formStore]);
```
