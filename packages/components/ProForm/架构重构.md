# ProForm 架构重构方案

## 一、背景：问题分析

### 1.1 当前架构

```
用户代码                          ProForm 组件
┌───────────────┐               ┌──────────────────────────┐
│ useProForm()  │               │  ProForm (forwardRef)    │
│               │               │                          │
│ instance A ───┼── 用到？ ──►  │  useProForm()  ← instance B
│ formStore A   │               │  formStore B             │
└───────────────┘               │  mergedSchemas           │
                                │  visibleSchemas           │
                                │  Grid + Buttons           │
                                │  <Provider>               │
                                └──────────────────────────┘
```

用户调用 `useProForm()` 创建了 `instance A` + `formStore A`，然后渲染 `<ProForm />`，组件内部又调用 `useProForm()` 创建了 `instance B` + `formStore B`。

**结果是两套独立的状态共存，互不相通。** 用户操作表单更新 `formStore A`，组件渲染依赖 `formStore B`，两个 store 各自独立变化。

### 1.2 职责交叉

`useProForm` 当前承担了太多不属于"纯数据层"的职责：

| 功能                                             | 正确归属 | 说明                                    |
| ------------------------------------------------ | -------- | --------------------------------------- |
| `formStore` 创建                                 | 数据层   | 表单数据仓库                            |
| `arcoForm` 创建                                  | 桥梁层   | Arco Form 实例，与 store 同生命周期     |
| `validate` / `setFieldsValue` / `getFieldsValue` | 数据层   | 操作表单数据                            |
| `fieldNavigation`                                | 渲染层   | DOM 键盘焦点管理、onKeyDown 处理器      |
| `virtualScroll`                                  | 渲染层   | 虚拟滚动渲染策略                        |
| `isDraftState` / `isPreviewState`                | 渲染层   | UI 模式状态                             |
| `fieldStatusMap`                                 | 渲染层   | 字段显隐/只读策略                       |
| `componentRefs` / `setComponentRef` / `getRef`   | 渲染层   | DOM 引用注册                            |
| `bindingProps`                                   | 渲染层   | props 再包装                            |
| `Provider`                                       | 渲染层   | React 组件创建                          |
| `scrollToField`                                  | 渲染层   | DOM 滚动操作                            |
| `schemas state`                                  | 渲染层   | 字段配置（setSchemas 改变布局而非数据） |

### 1.3 数据源分裂

`useProForm` 持有 `schemas state`（用 `useState` 保存），`ProForm` 又从这派生 `mergedSchemas` → `visibleSchemas`。用户通过 `setSchemas()` 更新的是 `useProForm` 的 state，然后通过 `bindingProps.schemas` 流到 `ProForm`。**数据源分裂在两层的边界上，没有唯一定义者。**

---

## 二、核心原则

### 原则 1：禁止重复 store

任何时候，一个表单实例只能有一个 `FormStore`。如果 `useProForm` 已创建，`ProForm` 必须复用，不得再创建。

### 原则 2：`useProForm` 拥有数据层，`ProForm` 拥有渲染层

```
useProForm → FormStore + arcoForm + 数据操作方法（validate / setValues / ...）
ProForm    → schemas state + Schema 合并 + 布局 + 按钮 + 草稿 + 字段导航
```

### 原则 3：`useProForm` 公开 API 不变

用户已有的 `const form = useProForm({...}); form.instance.validate()` 等全部照常工作。

### 原则 4：零破坏性变更

现有 `useProForm` + `<ProForm>` 独立使用保持完全向后兼容，只新增 `form` prop。

## 三、最终架构

### 3.1 总览

```
                   ┌──────────────────────────┐
                   │     useFormStore         │  ← 新增内部 hook
                   │     (内部，纯数据层)       │
                   │                          │
                   │ formStore + arcoForm     │
                   │ baseInstance:            │
                   │   validate               │
                   │   setFieldsValue         │
                   │   getFieldsValue         │
                   │   resetFields            │
                   │   submit                 │
                   └──────────┬───────────────┘
                              │ 组合
                   ┌──────────▼───────────────┐
                   │     useFormUI            │  ← 新增内部 hook
                   │     (内部，UI 能力层)      │
                   │                          │
                   │ fieldNavigation          │
                   │ fieldStatusMap           │
                   │ virtualScroll            │
                   │ draft/preview state      │
                   │ 扩展 instance 方法:       │
                   │   focusField             │
                   │   scrollToField          │
                   │   getFieldStatus         │
                   │   setFieldStatus         │
                   │   isDraft / setDraft     │
                   │   isPreview / setPreview │
                   │   getFieldFocused        │
                   └──────────┬───────────────┘
                              │ 组合
                   ┌──────────▼───────────────┐
                   │     useProForm           │  ← 公开 API，不变
                   │     (对外组合层)          │
                   │                          │
                   │ instance = { ...数据 + UI }│
                   │ bindingProps             │
                   │ ✗ Provider 不再返回       │
                   │ 保持 UseProFormReturn    │
                   └──────────────────────────┘
```

### 3.2 Provider 内置化

`ProFormContext.Provider` 从 `useProForm` 的返回值中移除，改为由 `ProForm` 内部创建。

**理由：** Provider 是纯渲染层产物（React 组件），不应当从数据层 hook 中暴露。`ProForm` 渲染时由其内部子组件（`ProFormWithForm` / `ProFormStandalone`）各自创建 Provider。

```tsx
// ProForm 内部：
const Provider = useMemo(() => {
  const P: React.FC<{ children: React.ReactNode }> = ({ children }) => (
    <ProFormContext.Provider value={{ formStore, instance, arcoForm }}>{children}</ProFormContext.Provider>
  );
  return P;
}, [formStore, instance, arcoForm]);
```

**对子组件无影响：** `useProFormContext()` 照常可用，因为 Provider 仍然在组件树中。
**对用户无影响：** 用户从来不需要手动操作 Provider，它一直是内部实现细节。

### 3.3 ProForm 内部调度

```tsx
// ProForm.tsx — 简化后的伪代码

export const ProForm = forwardRef((props, ref) => {
  // 调度层：选择数据源
  if (props.form) {
    return <ProFormWithForm form={props.form} {...props} ref={ref} />;
  }
  return <ProFormStandalone {...props} ref={ref} />;
});

// 受控模式：外部 useProForm 创建，ProForm 不重复创建
const ProFormWithForm = forwardRef(({ form: fullState, ...props }, ref) => {
  const { formStore, arcoForm, instance, setComponentRef, schemas,
          fieldNavigation, virtualState, virtualContainerRef,
          isDraftState, isPreviewState, setIsDraftState, setIsPreviewState } = fullState;

  // Provider 由 ProForm 内部创建，不从 fullState 取
  const Provider = useMemo(() => {
    const P: React.FC<{ children: React.ReactNode }> = ({ children }) => (
      <ProFormContext.Provider value={{ formStore, instance, arcoForm }}>
        {children}
      </ProFormContext.Provider>
    );
    return P;
  }, [formStore, instance, arcoForm]);

  useImperativeHandle(ref, () => instance, [instance]);

  return (
    <Provider>
      <ProFormRenderer
        {...props}
        formStore={formStore}
        arcoForm={arcoForm}
        instance={instance}
        setComponentRef={setComponentRef}
        fieldNavigation={fieldNavigation}
        virtualState={virtualState}
        virtualContainerRef={virtualContainerRef}
        isDraftState={isDraftState}
        isPreviewState={isPreviewState}
        setIsDraftState={setIsDraftState}
        setIsPreviewState={setIsPreviewState}
      />
    </Provider>
  );
});

// 独立模式：内部创建一次 useProForm
const ProFormStandalone = forwardRef((props, ref) => {
  const fullState = useProForm(props);
  const { formStore, arcoForm, instance, setComponentRef,
          fieldNavigation, virtualState, virtualContainerRef,
          isDraftState, isPreviewState, setIsDraftState, setIsPreviewState } = fullState;

  // Provider 由 ProForm 内部创建
  const Provider = useMemo(() => {
    const P: React.FC<{ children: React.ReactNode }> = ({ children }) => (
      <ProFormContext.Provider value={{ formStore, instance, arcoForm }}>
        {children}
      </ProFormContext.Provider>
    );
    return P;
  }, [formStore, instance, arcoForm]);

  useImperativeHandle(ref, () => instance, [instance]);

  return (
    <Provider>
      <ProFormRenderer
        {...props}
        formStore={formStore}
        arcoForm={arcoForm}
        instance={instance}
        setComponentRef={setComponentRef}
        fieldNavigation={fieldNavigation}
        virtualState={virtualState}
        virtualContainerRef={virtualContainerRef}
        isDraftState={isDraftState}
        isPreviewState={isPreviewState}
        setIsDraftState={setIsDraftState}
        setIsPreviewState={setIsPreviewState}
      />
    </Provider>
  );
});

// 纯渲染层：无状态创建 hooks，已处于 Provider 内
// Provider 由 ProFormWithForm / ProFormStandalone 包裹，ProFormRenderer 不需要感知
const ProFormRenderer: React.FC<ProFormRendererProps> = (props) => {
  const {
    formStore, arcoForm, instance, setComponentRef,
    fieldNavigation, virtualState, virtualContainerRef,
    isDraftState, isPreviewState, setIsDraftState, setIsPreviewState,
    schemas = [], layout, columns, ...rest
  } = props;

  // Schema 合并（纯 useMemo，不从 useProForm 取）
  const mergedSchemas = useMemo(() => mergeGlobalConfig(schemas, rest), [...]);

  // 懒加载 / 优先级加载（UI 优化策略）
  const visibleSchemas = useMemo(() => computeVisible(mergedSchemas, ...), [...]);

  // 草稿持久化（useEffect 副作用）
  useDraft({ formStore, ...draftConfig });

  // 折叠展开（UI 状态）
  const [collapsed, setCollapsed] = useState(defaultCollapsed);

  // Grid 布局 / 按钮组 / Context Providers
  // 所有渲染代码，没有 useProForm 调用
};
```

## 四、文件改动明细

### 4.1 文件：`useProForm.tsx`

**变化：**

1. 新增内部 hook `useFormStore`，从当前 `useProForm` 中提取纯数据部分
2. 新增内部 hook `useFormUI`，从当前 `useProForm` 中提取 UI 能力部分
3. `useProForm` 组合 `useFormStore` + `useFormUI`，保持现有 `UseProFormReturn` 接口和返回值形状不变

**`useFormStore`（内部，纯数据）：**

```typescript
// 入参：schemas 作为入参而非内部 state
function useFormStore(schemas: ProFormSchema[]) {
  const formStore = useMemo(() => createFormStore(), []);
  const arcoForm = useArcoForm(formStore);

  // 注册字段（schemas 变化时重新注册）
  useEffect(() => {
  const formStore = useMemo(() => createFormStore(), []);
  const arcoForm = useArcoForm(formStore);

  // 注册字段（schemas 变化时重新注册）
  useEffect(() => {
    schemas.forEach(schema => {
      const field = formStore.getField(schema.name);
      if (!field) {
        createFieldNode(schema, formStore);
      }
    });
  }, [schemas, formStore]);

  const validate = useCallback(async () => { ... }, [formStore, arcoForm]);
  const setFieldsValue = useCallback((values) => { ... }, [formStore, arcoForm]);
  const getFieldsValue = useCallback((nameList?) => { ... }, [formStore]);
  const resetFields = useCallback((nameList?) => { ... }, [formStore]);
  const submit = useCallback(async () => validate(), [validate]);
  const clearValidate = useCallback((name?) => { ... }, [formStore, arcoForm]);
  const getFieldValue = useCallback((name) => { ... }, [formStore]);
  const setFieldValue = useCallback((name, value) => { ... }, [formStore, arcoForm]);

  const baseInstance: Partial<ProFormInstance> = {
    validate, validateField, clearValidate,
    setFieldsValue, setFieldValue,
    getFieldValue, getFieldsValue,
    resetFields, submit,
  };

  return { formStore, arcoForm, baseInstance };
}

**`useFormUI`（内部，UI 能力）：**

```typescript
function useFormUI(
  schemas: ProFormSchema[],
  formStore: FormStore,
  arcoForm: ArcoFormInstance,
  options: { keyboardNavigation?, onFieldFocus?, onFieldBlur? }
) {
  const componentRefs = useRef<Record<string, unknown>>({});
  const [fieldStatusMap, setFieldStatusMap] = useState<Record<string, FieldStatus>>({});
  const [isDraftState, setIsDraftState] = useState(false);
  const [isPreviewState, setIsPreviewState] = useState(false);

  const getRef = useCallback((name: string) => componentRefs.current[name], []);
  const setComponentRef = useCallback((name: string, ref: unknown) => {
    componentRefs.current[name] = ref;
  }, []);

  const fieldNavigation = useFieldNavigation({
    schemas, getRef, keyboardNavigation, formStore,
    onFocus: options.onFieldFocus,
    onBlur: options.onFieldBlur,
  });

  const { containerRef: virtualContainerRef, virtualState, scrollToIndex } =
    useVirtualScroll(schemas, { ... });

  const scrollToField = useCallback((name: string) => {
    if (virtualState.visibleItems?.length) {
      const index = schemas.findIndex(...);
      if (index !== -1) { scrollToIndex(index); return; }
    }
    arcoForm.scrollToField(name);
  }, [arcoForm, schemas, scrollToIndex, virtualState]);

  const uiMethods = {
    focusField: fieldNavigation.focusField,
    focusNextField: fieldNavigation.focusNextField,
    focusPrevField: fieldNavigation.focusPrevField,
    getFocusedField: () => fieldNavigation.focusedField,
    scrollToField,
    getFieldStatus: (name: string) => fieldStatusMap[name] || 'edit',
    setFieldStatus: (name: string, status: FieldStatus) => {
      setFieldStatusMap(prev => ({ ...prev, [name]: status }));
    },
    isDraft: () => isDraftState,
    setDraft: (v: boolean) => setIsDraftState(v),
    isPreview: () => isPreviewState,
    setPreview: (v: boolean) => setIsPreviewState(v),
    getFieldFocused: (name: string) => formStore.getField(name)?.focused || false,
    getRef,
    setComponentRef,
  };

  return {
    componentRefs, setComponentRef, getRef,
    fieldNavigation, fieldStatusMap, setFieldStatusMap,
    isDraftState, setIsDraftState, isPreviewState, setIsPreviewState,
    virtualState, virtualContainerRef,
    uiMethods,
  };
}
```

**`useProForm`（组合层，API 不变）：**

```typescript
export function useProForm<TValues = Record<string, unknown>>(
  options: UseProFormOptions<TValues> = {},
): UseProFormReturn<TValues> {
  const { schemas: initialSchemas = [], ...rest } = options;

  // schemas state 保留（setSchemas 需要更新它）
  const [schemas, setSchemasState] = useState<ProFormSchema<TValues>[]>(initialSchemas);

  // 数据层
  const { formStore, arcoForm, baseInstance } = useFormStore(schemas);

  // UI 能力层
  const {
    uiMethods, setComponentRef, fieldNavigation,
    virtualState, virtualContainerRef,
    isDraftState, setIsDraftState, isPreviewState, setIsPreviewState,
    setFieldStatusMap, fieldStatusMap,
  } = useFormUI(schemas, formStore, arcoForm, { keyboardNavigation, onFieldFocus, onFieldBlur });

  // 合并 instance
  const instance: ProFormInstance<TValues> = useMemo(() => ({
    ...baseInstance,
    ...uiMethods,
    setSchemas: setSchemasState,
    setProps: (props) => setFormPropsState(prev => ({ ...prev, ...props })),
    getRef,
  }), [baseInstance, uiMethods, ...]);

  // bindingProps（保持原有形状）
  const bindingProps = useMemo(() => ({ schemas, ... }), [...]);

  // Provider 由 ProForm 内部创建，useProForm 不暴露

  return {
    arcoForm, instance, schemas, setSchemas: setSchemasState,
    formProps: initialFormProps,
    setComponentRef, fieldStatusMap, setFieldStatusMap,
    isDraftState, setIsDraftState, isPreviewState, setIsPreviewState,
    options, bindingProps, formStore,
    fieldNavigation, virtualState, virtualContainerRef,
  };
}
```

### 4.2 文件：`ProForm.tsx`

**变化：**

1. 新增 `form?: UseProFormReturn` prop
2. 新增 `ProFormWithForm` 内部组件（受控模式）
3. 新增 `ProFormStandalone` 内部组件（独立模式，行为与现在一致）
4. 提取 `ProFormRenderer` 纯渲染组件（不调用任何状态创建 hook）
5. 顶层 `ProForm` 根据是否有 `form` prop 分发到对应子组件

  formStore: FormStore;
  arcoForm: ArcoFormInstance;
  instance: ProFormInstance;
  setComponentRef: (name: string, ref: unknown) => void;
  fieldNavigation: UseFieldNavigationReturn;
  virtualState: VirtualScrollState<ProFormSchema>;
  virtualContainerRef: React.RefObject<HTMLDivElement | null>;
  isDraftState: boolean;
  isPreviewState: boolean;
  setIsDraftState: (v: boolean) => void;
  setIsPreviewState: (v: boolean) => void;

    formStore, arcoForm, instance, setComponentRef,
    fieldNavigation, virtualState, virtualContainerRef,
    isDraftState, isPreviewState, setIsDraftState, setIsPreviewState,
    Provider,
    schemas: rawSchemas = [], layout, columns,

  // Schema 合并（纯 useMemo）
  const mergedSchemas = useMemo(() => {
    return rawSchemas.map(schema => mergeGlobalConfig(schema, props));

  const [collapsed, setCollapsed] = useState(defaultCollapsed);

  return (
    <RootContextProvider value={rootContext}>
      <LayoutContextProvider value={layoutContext}>
        {renderFormContent(visibleSchemas, ...)}

  );

### 4.3 文件：`types.ts`

**变化：** 新增 `ProFormProps.form` 类型

````typescript
export interface ProFormProps<TValues = Record<string, unknown>> {
  /**
   * 表单状态对象（由 useProForm 返回）。
   *
   * 传入后 ProForm 复用该状态，不再内部调用 useProForm 创建新实例。

   * <ProForm schemas={[...]} ref={formRef} />
   * // formRef.current 获取实例
   * ```

}

## 五、使用方式对比

### 5.1 方式 A：独立使用（不变）

```tsx
import { ProForm, type ProFormInstance } from '@lania-pro-components/components';

function MyForm() {
  const ref = useRef<ProFormInstance>(null);

  return (
    <ProForm
      ref={ref}
      schemas={[
        { name: 'name', label: '姓名', component: 'Input' },
        { name: 'email', label: '邮箱', component: 'Input' },
      ]}
      columns={2}
      onFinish={(values) => console.log(values)}
    />
  );
}
```

`ref.current` 包含完整实例（validate / getValues / focusField / scrollToField 等全部方法）。

### 5.2 方式 B：配合 `useProForm`（推荐，解决重复）

```tsx
import { ProForm, useProForm } from '@lania-pro-components/components';

function MyForm() {
  const form = useProForm({
    schemas: [
      { name: 'name', label: '姓名', component: 'Input' },
      { name: 'email', label: '邮箱', component: 'Input' },
    ],
    initialValues: { name: '张三' },
  });

  const handleSubmit = async () => {
    const values = await form.instance.validate();
    // 编程式访问表单
    form.instance.focusField('name');
    form.instance.scrollToField('email');
  };

  return (
    <div>
      <ProForm form={form} columns={2} />
      {/* form.instance 是唯一的，ProForm 内部直接复用 */}
      <button onClick={handleSubmit}>提交</button>
    </div>
  );
}
```

### 5.3 方式对比

|              | 方式 A（ref）    | 方式 B（form prop）        |
| ------------ | ---------------- | -------------------------- |
| 获取实例方式 | `ref.current`    | `form.instance`            |
| 实例数量     | 1 个（内部创建） | **1 个**（外部传入）       |
| 推荐度       | ⭐⭐⭐ 简单场景  | ⭐⭐⭐ 需要编程控制        |
| 适用场景     | 仅有表单提交     | 动态字段/值控制 + 复杂交互 |

## 六、变更影响

### 6.1 破坏性变更

**无。** 所有现有 API 保持完全向后兼容：

- `useProForm` 的返回值形状不变（仅移除 `Provider`，用户代码从不直接使用它）
- `ProForm` 的 props 保持向后兼容
- `ProFormContext` / `useProFormContext` 保持可用（Provider 由 ProForm 内部渲染）

### 6.2 新增导出

`ProFormProps.form` 类型（`UseProFormReturn`）。

### 6.3 推荐迁移路径

1. 新代码统一使用方式 B（`form` prop）
2. 旧代码按需迁移，无需强制
